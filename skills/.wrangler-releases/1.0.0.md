# Wrangler v1.0.0 - Initial Release

## Overview

Initial release of wrangler governance framework and skills library.

## Features

### Skills Library (36 skills)
- **Testing**: TDD, async testing, anti-patterns, condition-based waiting
- **Debugging**: Systematic debugging, root cause tracing
- **Collaboration**: Planning, code review, parallel agents, subagent-driven development
- **Governance**: Constitution, roadmap, issue tracking, verification
- **Meta**: Writing skills, testing skills with subagents

### Built-in MCP Server
- 11 issue management tools
- Markdown-based issue and specification storage
- Automatic workspace initialization
- Full-text search and metadata queries
- 233 comprehensive tests with 87.11% coverage

### Governance System
- Constitutional principles for consistent decision-making
- Hierarchical planning (strategic roadmaps to tactical execution)
- Systematic issue and specification tracking
- Automated governance verification and maintenance

### Workflow Integration
- Slash commands for common operations (/brainstorm, /write-plan, /execute-plan)
- Session hooks for automatic setup
- Verification and validation tools
- Metric tracking and status reporting

## Directory Structure (v1.0.0)

```
project-root/
├── issues/                    # Root-level issue tracking
├── specifications/            # Root-level specifications
│   ├── _CONSTITUTION.md       # If governance initialized
│   ├── _ROADMAP.md
│   └── _ROADMAP__NEXT_STEPS.md
└── memos/                     # Root-level reference material
```

## Known Limitations

- No versioning system (projects don't track wrangler version)
- Governance files scattered across root directories
- No automatic migration for breaking changes
- No cache configuration for MCP server
