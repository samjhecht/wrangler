# Definition of Done

> A task is considered "done" only when ALL applicable criteria are met.

## Code Quality

- [ ] Code is written and compiles without errors
- [ ] Code follows project style guide and conventions
- [ ] No commented-out code or debug statements
- [ ] Complex logic has explanatory comments
- [ ] Functions/methods are appropriately sized (prefer < 50 lines)
- [ ] DRY principle followed (no unnecessary duplication)

## Testing

- [ ] Tests written using TDD (test-first approach)
- [ ] Unit tests cover happy path and edge cases
- [ ] Integration tests cover component interactions (if applicable)
- [ ] All tests pass locally
- [ ] Test coverage meets project minimum (typically 80%+)
- [ ] No flaky tests introduced

## Git Hooks Compliance

- [ ] Pre-commit hooks pass (format, lint, unit tests)
- [ ] Pre-push hooks pass (full test suite on protected branches)
- [ ] Commit messages follow project conventions
- [ ] No bypass used (or bypass is documented with justification)

## Documentation

- [ ] Code is self-documenting with clear naming
- [ ] Public APIs have documentation/docstrings
- [ ] README updated for new features (if applicable)
- [ ] Architecture decisions documented (if significant)
- [ ] Breaking changes documented

## Security

- [ ] No secrets or credentials in code
- [ ] Input validation for user-provided data
- [ ] Authorization checks where required
- [ ] Dependencies checked for known vulnerabilities
- [ ] Security checklist reviewed

## Review

- [ ] Self-review completed
- [ ] Code review requested and approved
- [ ] All review comments addressed
- [ ] CI/CD pipeline passes

## Deployment Readiness

- [ ] Feature flags configured (if applicable)
- [ ] Database migrations work (if applicable)
- [ ] Environment variables documented
- [ ] Rollback plan exists (for critical changes)

## Accessibility (for UI changes)

- [ ] Keyboard navigation works
- [ ] Screen reader compatible
- [ ] Color contrast meets WCAG guidelines
- [ ] Form inputs have labels

## Performance (if applicable)

- [ ] No obvious performance regressions
- [ ] Large data sets handled efficiently
- [ ] Caching considered where appropriate
- [ ] Lazy loading for heavy resources

---

## Quick Reference

### Minimum Requirements (all tasks)

1. Code compiles and works
2. Tests pass
3. Git hooks pass
4. Code review approved

### Additional for Features

5. Documentation updated
6. Security checklist reviewed

### Additional for Releases

7. Deployment verified
8. Monitoring in place
